DB-앱 연동 안내 마법사

컴퍼스랩 · v0.app으로 만든 앱을 스프레드시트와 연결합니다 · 약 20분
9단계 중 1단계 진행 중

시작하기

5장에서 만든 앱을 구글 스프레드시트와 연결합니다. 진짜 데이터를 쓰는 앱이 됩니다.

이 마법사가 도와드리는 것
복잡한 7단계 절차를 화면 하나씩 따라가며, 자동으로 만들어주는 부분(스프레드시트 ID 추출, 앤비 프롬프트 생성, .env.local 파일 다운로드)은 클릭으로 끝냅니다.

준비물 3가지

1

구글 계정

구글 클라우드 콘솔과 스프레드시트에 로그인할 계정.

2

v0.app으로 만든 앱 폴더

5장에서 만든 Next.js 앱이 들어 있는 폴더 (앤티그래비티에서 열 수 있는 상태).

3

스프레드시트 파일

샘플(FranklinDiary)을 쓸 분은 강사가 보내준 파일, 본인 DB로 진행할 분은 본인 스프레드시트.

1단계 — DB 종류 선택

앱과 연결할 데이터 저장소를 고릅니다.

📊

구글 스프레드시트

가장 쉽고 빠릅니다.
이 마법사가 안내합니다.

🗃️

Supabase (준비 중)

큰 데이터·로그인 기능이 필요할 때.
다음 버전에서 지원 예정.

2단계 — 모드 선택

처음이면 샘플로, 익숙하면 본인 DB로 진행하세요.

🧪

샘플 (FranklinDiary)

강사가 준비한 "할 일" 시트로
처음부터 끝까지 따라합니다.

📁

내 DB로

본인이 만든 스프레드시트로
그대로 진행합니다.

3단계 — 구글 클라우드 콘솔 설정

앱이 스프레드시트로 들어갈 수 있는 "허가증 발급소"를 엽니다.

🏛️ 비유: 구글 클라우드 콘솔은 시청 민원실입니다. 여기서 우리 앱이 사용할 허가증을 발급받습니다.
1

콘솔 접속

console.cloud.google.com 접속해서 구글 계정으로 로그인

2

새 프로젝트 만들기

좌상단 프로젝트 드롭다운 → [새 프로젝트] → 이름 입력(예: my-sheet-app) → 만들기

3

API 두 개 활성화 (중요)

좌측 메뉴 [APIs & Services] → [Library] → Google Sheets API와 Google Drive API를 각각 검색하여 [사용 설정(Enable)] 누르기.
두 개 모두 켜야 합니다.


4단계 — 서비스 계정 + JSON 키

앱 전용 허가증과 거기에 찍는 도장을 만듭니다.

🪪 비유: 서비스 계정은 우리 앱 전용 허가증이고, 🔏 JSON 키 파일은 거기에 찍는 도장입니다.
1

서비스 계정 만들기

[APIs & Services] → [Credentials] → [+ Create Credentials] → [Service account]
이름 입력(예: sheet-bot) → [Create and continue] → 권한은 비워두고 [Done]

2

이메일 복사

방금 만든 서비스 계정을 클릭 → Email 항목에 보이는 xxx@yyy.iam.gserviceaccount.com 형태의 주소를 메모장에 복사.
(다음 화면에서 스프레드시트 공유에 씁니다.)

3

JSON 키 다운로드 (도장 받기)

같은 화면에서 [Keys] 탭 → [Add Key] → [Create new key] → 종류 JSON → [Create]
→ 파일이 자동으로 다운로드됩니다 (service_account.json 비슷한 이름).

⚠️ JSON 파일 보안 경고
이 파일은 앱의 도장입니다. 누군가 이 파일을 가지면 여러분 앱처럼 행동할 수 있습니다.
• 절대 카톡·메일·깃허브에 올리지 마세요.
• 이 마법사 어디에도 JSON 내용을 입력하지 마세요. (이 마법사는 절대 묻지 않습니다.)
• 본인 컴퓨터에만 두고, .env.local에 값만 옮겨 적습니다.

5단계 — 스프레드시트 공유 + ID 추출

앱이 들어갈 시트를 공유하고, 시트 ID를 자동으로 뽑아냅니다.

🔑 비유: 시청에서 허가증을 받았어도, 우리 집에 들어오려면 "우리 집 방문을 허락"해야 합니다. 그게 시트 공유입니다.
1

스프레드시트 공유 (편집자 권한)

구글 드라이브에서 시트를 열고 → 우상단 [공유] → 4단계에서 복사한 서비스 계정 이메일(xxx@...iam.gserviceaccount.com)을 붙여넣고 → 권한을 편집자(Editor)로 → [보내기].

2

URL 붙여넣고 ID 뽑기

시트 주소창의 URL 전체를 복사해서 아래 칸에 붙여넣으세요. ID를 자동으로 뽑아드립니다.

여기에 추출된 ID가 표시됩니다.
이 ID는 다음에 만들 .env.local의 GOOGLE_SPREADSHEET_ID 값입니다.

6단계 — .env.local 템플릿 다운로드

비밀 정보를 담을 "금고"의 빈 양식을 받아갑니다.

🔐 비유: .env.local은 집 안의 금고입니다. 외부에 절대 나가지 않고 본인 컴퓨터에만 있습니다.
✅ 이 다운로드 버튼은 인터넷에 어떤 정보도 보내지 않습니다. 빈 양식만 만들어 줍니다. JSON 키 값은 본인이 직접 메모장에서 옮겨 적으세요.

받은 후 할 일

1

파일 위치

다운받은 파일을 v0.app 앱 폴더의 최상단(package.json이 있는 폴더)에 복사하세요.

2

빈칸 3개 채우기

메모장으로 .env.local 열고:
• GOOGLE_SERVICE_ACCOUNT_EMAIL = JSON 안의 client_email 값
• GOOGLE_PRIVATE_KEY = JSON 안의 private_key 값 (큰따옴표 그대로)
• GOOGLE_SPREADSHEET_ID = 5단계에서 뽑은 ID

3

주의 (오타 1순위)

• 등호(=) 앞뒤에 공백 넣지 않습니다.
• GOOGLE_PRIVATE_KEY의 \n은 그대로 둡니다 (실제 줄바꿈으로 바꾸면 오류).
• 큰따옴표(") 절대 빼먹지 마세요.

7단계 — 앤티그래비티 프롬프트 만들기

시트 구조만 입력하면, 앤비에 그대로 붙여넣을 프롬프트를 자동으로 만들어 드립니다.

복사 후 앤티그래비티 대화창에 그대로 붙여넣으세요.
다음 단계 (앤비에서)
앤비가 [Run] 버튼을 띄우면 순서대로 클릭하세요. 앱이 수정되면 다음 문장을 추가로 입력해 로컬에서 확인합니다:
로컬호스트에서 네가 수정한 앱을 보여줘.

8단계 — Vercel 환경변수 + 오류 대응

로컬에서 작동을 확인했다면, Vercel(앱을 24시간 켜두는 곳)에도 같은 비밀을 넣어줍니다.

☁️ 비유: Vercel은 외부 서버입니다. 본인 컴퓨터에 있는 .env.local을 자동으로 옮겨주지 않으므로, 같은 변수를 Vercel에도 직접 넣어줘야 합니다.

Vercel 환경변수 추가 (3단계)

1

대시보드 진입

vercel.com/dashboard → 해당 프로젝트 선택 → 상단 [Settings] → 좌측 [Environment Variables]

2

변수 3개 추가

아래 3개를 한 줄씩 추가합니다. 값은 .env.local과 완전히 동일해야 합니다.
• GOOGLE_SERVICE_ACCOUNT_EMAIL
• GOOGLE_PRIVATE_KEY  ("..." 큰따옴표 포함, \n 그대로)
• GOOGLE_SPREADSHEET_ID

3

Redeploy

상단 [Deployments] → 가장 최근 배포 옆 […] → [Redeploy]. 환경변수가 반영됩니다.

오류가 나면? — 자주 나오는 5가지

❶ "The caller does not have permission" — 권한 없음
원인: 스프레드시트에 서비스 계정 이메일이 공유되지 않음.
해결: 5단계로 돌아가서 시트 [공유]에 xxx@...iam.gserviceaccount.com을 편집자(Editor)로 추가했는지 확인.
❷ "DECODER routines::unsupported" / PEM 오류
원인: GOOGLE_PRIVATE_KEY 값에서 \n이 훼손되었거나 큰따옴표가 빠짐.
해결: .env.local 템플릿을 다시 다운로드해서, JSON의 private_key 전체를 큰따옴표로 감싸 붙여넣기. \n은 그대로 두세요.
❸ "Requested entity was not found" — 시트 못 찾음
원인: 시트 탭 이름 오타 (대소문자·공백·한글 포함).
해결: 시트 하단 탭 이름을 그대로 복사해서 코드·프롬프트에 붙여넣기.
❹ "Sheets API has not been used" — API 안 켰음
원인: 3단계에서 Sheets API 또는 Drive API 활성화 빠뜨림.
해결: 3단계로 돌아가서 두 API 모두 [사용 설정] 확인.
❺ 로컬은 되는데 Vercel만 안 됨
원인: Vercel에 환경변수 누락 또는 Redeploy 안 함.
해결: 위 Vercel 3단계 다시 확인. 변수 추가 후 반드시 Redeploy.
🎉 여기까지 오셨다면 연동 완료입니다!
